# Anima LoRA Block Weight — 使用说明书（中文）

> 一个 ComfyUI 节点：**Anima LoRA Block Weight**（菜单分类 `loaders`）
> 适用于 Anima（NVIDIA Cosmos 架构，2B 二次元模型）的 LoRA。
> 节点支持两种模式：三档分层（grouped）与逐 block 精细控制（per_block）。

本文档分两部分：
- [Part A · 小白向](#part-a小白向看这部分就能上手) — 用大白话和傻瓜配方手把手教。
- [Part B · 技术向](#part-b技术向原理参数与实现) — 原理、完整参数、实现细节。

---

## Part A·小白向（看这部分就能上手）

### 1. 这个节点帮你做什么

一个 LoRA 就像一份"混了好几种味道的调料包"，里面同时含有：
- **画风**（线条、上色、笔触）
- **构图**（站位、镜头、画面布局）
- **骨相 / 体型**（脸型、身材、姿势倾向）

普通 LoRA 加载器只有一个"总开关"，没法单独取舍。这个节点给你一套**分开的小开关**，
让你能"只要画风、不要它自带的身材"，或者反过来。

它专为 Anima 设计。网上现成的同类节点是给别的模型（Flux 等）写的，装到 Anima 上调了
没反应——这个节点解决了那个问题。

### 2. 一个核心常识：前段管骨架，后段管画风

Anima 内部有 28 个"处理段"（编号 0–27，称为 block）。记住这个规律就够：

| 段 | 编号 | 管什么 |
|----|------|--------|
| 前段 | 0–8 | **骨架**：构图、姿势、体型、骨相 |
| 中段 | 9–18 | 过渡，介于两者之间 |
| 后段 | 19–27 | **皮肤**：画风、上色、纹理 |

口诀：**前段画骨架，后段画皮肤。** 想改身材姿势就动前段，想改画风就动后段。

数值含义：`1.0` = 原样保留；`0.5` = 影响减半；`0.0` = 完全关掉；`1.5` = 加强（别超 1.5，易崩）。

> ⚠️ 上面的分界（0–8 / 9–18 / 19–27）是经验上的"大概"，你这个 LoRA 的真实分界可能略有出入，
> 见 Part B 的"扫描定位法"测出来。

### 3. 怎么用（三档模式，最简单）

1. 把节点接进工作流，`control_mode` 选 **grouped**，选好你的 LoRA。
2. 先全部保持 `1.0`，出一张图当"原样对照"。
3. 想削弱哪方面，把对应开关往下调：
   - `shallow_weight` = 前段（构图骨相）
   - `middle_weight` = 中段
   - `deep_weight` = 后段（画风）
4. 出图对比。

### 4. 傻瓜配方（直接抄）

**配方1 — 要画风，不要它的身材/姿势**
```
shallow_weight = 0.3   middle_weight = 0.7   deep_weight = 1.0
```

**配方2 — 要姿势构图，换掉画风**
```
shallow_weight = 1.0   middle_weight = 0.7   deep_weight = 0.3
```

**配方3 — 画风想更浓**
```
shallow_weight = 1.0   middle_weight = 1.0   deep_weight = 1.2
```

**配方4 — 整体想减弱** → 不用分层，直接把 `strength_model` 调到 0.6 即可。

**只想削"体型/骨相"这一项、画风构图都保留** → 三档全留 1.0，只把 `w_self_attn` 调到 0.5。

### 5. 那四个 w_ 开关到底改的是什么（人话版）

节点上还有四个开关：`w_self_attn`、`w_cross_attn`、`w_mlp`、`w_adaln`。
它们看起来很玄，其实可以这样理解——

**把模型想象成一个画师**，这四个开关对应画师脑子里四种不同的活动：

- **`w_self_attn` = 画师在安排"画面里各部分的位置关系"**
  手接在肩膀下、眼睛对称、人站哪儿、构图怎么摆……这是画面内部各部分怎么协调。
  → 管：**构图、姿势、人体结构、骨相、体型**。
  → 骨相/身材过了想收，就调低它。这是你最常用的一个。

- **`w_cross_attn` = 画师在"读你的要求并照着画"**
  你说"穿红裙子的女孩"，画师得听懂并对应到画面。这是文字提示和画面之间的桥。
  → 管：**模型对提示词的响应**。
  → LoRA 老抢戏、或某个概念（比如漏标的蓝裤子）粘在触发词上想对冲，调低它。
    但副作用是它对你所有提示词都会变迟钝，是把"钝刀"不是"手术刀"。

- **`w_mlp` = 画师的"画功和风格"**
  怎么上色、笔触什么质感、线条粗细。和画什么内容无关，是"怎么画"。
  → 管：**画风、上色、纹理、笔触**。
  → 想加强或减弱画风本身，动它。

- **`w_adaln` = 画师的"整体状态/情绪"**
  今天整体下笔重还是轻、画面偏冷还是偏暖。是笼罩全局的一层微妙基调。
  → 管：**整体氛围、对比**（很微妙，一般感知不明显）。
  → **建议保持 1.0 不动**，等前三个玩熟了再说。

**它和前/中/后段是什么关系？**
前/中/后段是"第几道工序"（浅层粗胚、深层精修）；
这四个开关是"每道工序里用哪只手"。两者相乘。
比如 `deep_weight=1.0` 但 `w_mlp=0.5` = "后段工序整体保留，但后段里'画风那只手'减半"。

**新手期你其实只需常用一个：`w_self_attn`**（收骨相/身材）。
其余三个，`w_mlp` 偶尔调画风时用，`w_cross_attn` 遇概念污染时用，`w_adaln` 先忘掉它。

**想一次就记住区别？** 固定种子出三张图对比：
全 1.0 一张、只把 `w_self_attn` 调 0.3 一张、只把 `w_mlp` 调 0.3 一张。
你会看到一张"人变了画风没变"、一张"画风变了人没变"——一次就懂。

### 6. 出问题怎么办

- **调了没变化**：确认改的是 weight 数值而非区间文字；打开 `verbose` 看后台是否真的应用了；
  检查是否被另一个普通加载器覆盖。
- **画面崩坏 / 杂纹**：某个开关太高（超 1.5）或一边归零另一边总强度又高。先全部回 1.0 再慢慢偏离。
- **菜单里找不到节点**：确认文件放对、重启过 ComfyUI。

## Part B·技术向（原理、参数与实现）

### 1. 为什么需要这个节点

Anima 基于 NVIDIA Cosmos 架构，其 LoRA 的权重 key 命名为：

```
lora_unet_blocks_{N}_<submodule>...    # N = 0..27，下划线分隔
```

社区现有的 LoRA Block Weight 节点为 Flux/SDXL 设计，正则写死了点号命名
（`blocks\.(\d+)` 匹配 `blocks.0`），无法匹配 Anima 的下划线命名，导致所有张量
落入"未匹配"分支被乘以平均权重——分层完全失效。

本节点使用 `blocks[_.](\d+)` 正则，点号与下划线均可匹配。经真实 Anima LoRA
（28 block、896 个权重张量）验证：100% 匹配，零漏匹配。

### 2. 缩放原理

每个权重张量最终缩放系数：

```
factor = block权重 × 子模块类型系数
```

- **block 权重**：
  - `grouped` 模式来自 shallow / middle / deep 三档；
  - `per_block` 模式来自 `block_weights` 文本框逐 block 指定。
- **子模块类型系数**：self_attn / cross_attn / mlp / adaln 各一个乘数，两种模式下均生效。
- `.alpha` 张量不缩放（缩放 up/down 已等效改变贡献）。
- 加权后的张量通过 `comfy.sd.load_lora_for_models` 标准接口加载。

### 3. 层与子模块的职责

| 维度 | 取值 | 职责 |
|------|------|------|
| 深度 · 浅层 | block 0–8 | 全局构图、姿态、骨架、比例 |
| 深度 · 中层 | block 9–18 | 主体形态、语义过渡 |
| 深度 · 深层 | block 19–27 | 风格、笔触、纹理、上色 |
| 类型 · self_attn | `w_self_attn` | 图像内部空间结构、构图、骨相 |
| 类型 · cross_attn | `w_cross_attn` | 文本→画面的语义映射、提示词响应 |
| 类型 · mlp | `w_mlp` | 风格、纹理、画风特征 |
| 类型 · adaln | `w_adaln` | DiT 自适应归一化调制，整体调性 |

### 4. 完整参数表

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| model / clip | — | — | 接入模型与 CLIP |
| lora_name | 下拉 | — | 选择 LoRA 文件 |
| strength_model | float | 1.0 | LoRA 对模型的总强度 |
| strength_clip | float | 1.0 | LoRA 对 CLIP 的总强度 |
| control_mode | 选项 | grouped | `grouped` 三档 / `per_block` 逐块 |
| shallow_blocks / _weight | str / float | "0-8" / 1.0 | 浅层区间与系数（grouped） |
| middle_blocks / _weight | str / float | "9-18" / 1.0 | 中层区间与系数（grouped） |
| deep_blocks / _weight | str / float | "19-27" / 1.0 | 深层区间与系数（grouped） |
| block_weights | 多行文本 | "" | 逐 block 权重（per_block） |
| w_self_attn / w_cross_attn / w_mlp / w_adaln | float | 1.0 | 子模块类型系数 |
| default_weight | float | 1.0 | 未指定 block 的缺省系数 |
| verbose | bool | false | 控制台打印每 block 实际系数 |

### 5. per_block 文本框写法

三种写法可混用（逗号、换行、分号均可分隔），后写覆盖先写：

```
纯序列：   1,1,0.5,0.5            # 依次对应 block 0,1,2,3...；不足补 default
索引:值：  5:0.3, 27:1.2          # 仅改 block 5 和 27
区间:值：  0-8:0.3, 19-27:1.0     # 批量设区间
混写：     0-27:1.0, 12:0, 13:0   # 全设 1.0，再单独关掉 12、13
```

区间写法（shallow/middle/deep_blocks 与 block_range）支持 `all` / `0-8` / `0,3,5` / `0-8,19-27`。
区间重叠时优先级 deep > middle > shallow。

### 6. 扫描定位法（找出你 LoRA 的真实分界）

逐 block 模式最大价值是为你的 LoRA 画"功能地图"：

1. 固定 seed、提示词、采样器、步数、CFG（控制变量）。
2. 用 per_block，一次只关一小段：
   ```
   0-27:1.0, 0-3:0      # 只关 0~3
   0-27:1.0, 4-7:0      # 只关 4~7
   ...                  # 直到 24-27
   ```
3. 每段对比"全 1.0 原图"：关掉后**构图塌了** → 该段管骨架；**画风变了** → 该段管画风。
4. 记录结果，回到 grouped 模式按测出的真实分界设三档，日常高效。

### 7. 调参方法论

- 一次只动一组旋钮，固定种子做对比。
- 系数主要在 0.0–1.0 探索；>1.0 为放大，易出伪影，上限 2.0。
- 打开 verbose 核对实际施加值与预期一致。

### 8. 安装

```
ComfyUI/custom_nodes/anima-lora-block-weight/
    ├── __init__.py
    └── anima_lora_block_weight_pro.py
```

重启 ComfyUI，在 `loaders` 分类下找到节点。无第三方依赖。

### 9. 重要说明

文档中的 block 区间与数值为 DiT 通用经验起点，**非针对特定 LoRA 的实测值**。
张量分层逻辑已用真实 Anima LoRA 验证；"加载进模型后出图正常"需在实际环境确认。
