# Anima 画风 LoRA 分层行为研究记录

> 本文记录一系列在 Anima(Cosmos 架构,28 个 DiT block)画风 LoRA 上进行的分层消融实验,
> 目的是搞清楚 LoRA 的"功能信息"在 block 深度与子模块类型两个维度上如何分布,
> 以及它能否被分层节点干净地拆分控制。
>
> **所有测试均在固定种子、固定提示词、固定采样器/步数/CFG、关闭其它 LoRA 的单一变量条件下进行。**

---

## 一、测试对象

| LoRA | 训练目标 | adaln_modulation | network_dim/alpha | 优化器 | LR | 训练量 |
|------|----------|------------------|-------------------|--------|-----|--------|
| ks_v5-3 (主测) | 画风 (ks_style) | **有 (504 张量)** | 32 / 32 | AdamW | 1e-4 | 82图×1200步(测600) |
| firedotinc | 画风 | 无 | 16 / 16 | AdamW8bit | 5e-5 | 117图×1170步 |
| BlueArchiveStyleB1 | 画风 | 无 | 48 / 48 | AdamW | 1e-5 | 499图×7485步 |

三个 LoRA 训练配置差异极大(LR 相差 10 倍、dim 相差 3 倍、训练量相差 6 倍、有无 adaln),
是一组很强的交叉对照。

---

## 二、核心发现

### 1. 信息高度集中在 middle 段,且为架构级现象

三个 LoRA 尽管训练配置天差地别,分层行为**完全一致**:信息密度集中在中段,
shallow(0-8 附近)与 deep(19-27)单独影响都较小。

由于训练超参与 adaln 的有无都不改变这个结论,因此它**不是某个 LoRA 的训练产物,
而是 Anima 架构处理"画风"这类信息的固有方式**。

> ⚠ 该结论目前仅在画风 LoRA 上验证。角色类 LoRA 的分布逻辑可能不同,尚待验证。

### 2. adaln_modulation 不影响分层行为

ks_v5-3 额外包含 504 个 `adaln_modulation_*` 张量(训练时显式 include),
而另两个 LoRA 完全没有。但三者分层行为一致 —— 说明 adaln 的有无**不改变分层可控性**。
最初"adaln 导致无法细分层"的假设被证伪。

### 3. 子模块类型维度的实测分工

| 子模块系数 | 实测作用 |
|-----------|----------|
| `w_mlp` | 对骨相影响最大,同时影响画风(最强的单一维度开关) |
| `w_self_attn` | 细节内容 + 少量骨相 + 少量画风 |
| `w_cross_attn` | 对画风影响最小,适合安全微调 |
| `w_adaln` | 整体性影响,所有维度都会动,无专门方向 |

注:这些与 DiT 通用经验默认值差异很大(通用经验认为 self_attn 管空间结构、mlp 管画风),
属于该架构 / 该类 LoRA 的实测偏移。

### 4. middle 段内部存在梯度分工(非干净分区)

最初用"单段消融"(`0-27:1.0, 9-10:0` 形式,只关 2 个 block)测 middle 内部,
发现各段影响"平均且不明显"。后判明原因:**单段消融幅度太小(只关 2/28),
middle 内部信息冗余,被未关闭的相邻 block 补偿掉了**。

改用大幅度方法(方法A:只保留某段;方法B:整段关闭)后,梯度分工显现:

| block 段 | 实测倾向 | 关闭后的主要表现 |
|----------|----------|------------------|
| 0-11 | 体型整体尺寸的提示词服从度 | 关→体型听话缩放、姿势趋训练集固化、材质几乎不动 |
| 12-14 | 体型比例/敦实度的先验服从度 | 关→比例锁回先验、不听 petite 类提示词、面部基本不变 |
| 15-18 | LoRA 核心表达段 (骨相+比例+材质,信息密度最高) | 关→骨相比例材质全变、明显偏向底模输出 |
| 19-27 | 全局精修 (饱和度/纯度等表层属性) | 关→各维度平均小幅变化、颜色更纯 |

### 5. 最重要的机制结论:block 段控制的是"先验博弈权重",不是"画笔"

通过对抗性测试(用底模强先验角色"加贺" + 与之冲突的提示词 `petite body`/`shortstack figure`)
与原创弱先验角色对照,发现:

- 关闭某段 = **解除 LoRA 在该维度对抗底模/提示词的推力** → 结果回归"先验"
  - 底模有强先验角色时(加贺)→ 回归底模官方设定(成熟体型、官方服饰)
  - 原创弱先验角色时 → 回归 LoRA 训练集倾向(丰满体型)
- 因此各段的本质是**"该维度上,听提示词/LoRA vs 回归底模先验"的权重旋钮**,
  而非"负责画某个部位"。

这解释了为什么无法做到"干净分离":骨相、画风等并非"住在"某段,
而是以"先验偏离强度"的形式分布在多段中,彼此串扰。

### 6. 颜色实验佐证博弈框架(固定种子)

以画面中一个未被提示词定义的游泳圈为单一可量化变量(种子锚定其基础色域为红色系):

- 原图:红 → 关 0-11:黄 → 关 12-14:橙 → 关 15-18:橙红 → 关 19-27:更纯的红
- 关 0-11/12-14/15-18:色相在红色邻域内朝黄方向递减偏移(LoRA 各段在激活时"把颜色往红推",
  关掉即解除推力)
- 关 19-27:不旋转色相,而是往饱和度/纯度方向走 —— 印证其"表层精修"定位
- 全程不跨色域(不会变蓝),说明种子锚定了基础色,block 权重只在其上做有限调整

## 三、对工具设计的结论

1. **不需要"自适应识别分布形态"**:画风 LoRA 分布稳定且架构级一致,可直接内置经验区间。
2. **middle 值得拆分**:0-11 / 12-14 / 15-18 / 19-27 四段各有侧重,旧版"middle 一个旋钮"过粗。
3. **命名要诚实**:不应命名为"骨相/材质/动作"(暗示可干净分离),
   应采用中性段位命名 + 文档说明实测倾向与已知串扰,或采用"服从度"语义。
4. **区间应可自定义**:子段精细边界目前主要在 ks_v5-3 上验证,
   不同 LoRA 可能微调,故新版应预填实测默认值但允许用户修改。
5. **adaln 无需特殊处理**:保留为普通子模块系数即可,文档注明其对分层行为无显著影响。

---

## 四、方法论备注

- 单段消融幅度过小会被冗余补偿掩盖真实分工 —— 需用"只保留某段"或"整段关闭"等大幅度操作。
- 测试"功能"时必须控制底模先验强度:强先验角色与弱先验角色会让同一段表现出不同的可见效果,
  其实是同一"先验博弈"机制的不同侧面。
- 颜色等敏感维度建议多种子复现以排除随机性(本研究颜色结论基于固定种子下的单组观察,
  作为佐证而非主证)。
