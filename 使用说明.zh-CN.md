# Anima LoRA Block Weight

> **关于本项目**：这个插件是我在与 **Claude（Anthropic 的 AI）** 进行简单测试和深度讨论后，
> 由 Claude 开发的，而不是我本人编写的。分层逻辑、算法和代码都由 Claude 完成，我负责在真实
> 环境中测试、提供实验数据和反馈。UI 设计参考了
> [comfyUI-Realtime-Lora](https://github.com/shootthesound/comfyUI-Realtime-Lora)
> （逐层滑块 + impact 染色的交互思路）。特此说明，以免大家产生疑问或歧义。
>
> **界面语言**：节点界面文字会根据 ComfyUI 的语言设置（`Comfy > Locale`）自动在中文/英文
> 之间切换。切换的只是显示文字，后端参数名不变，所以工作流存储、API 调用不受影响。

为 LoRA 提供分层（block weight）控制的 ComfyUI 节点。最初为 **Anima**（NVIDIA Cosmos 架构的
2B 二次元模型）开发，现已发展成一个更通用的「按实测指标自动分段」的分层工具。既能在生成时实时
调整每层强度，也能把调好的分层烘焙成新的 LoRA 文件。

社区现有的 LoRA 分层节点多为 Flux/SDXL 设计，正则写死了点号命名（`blocks.0`），匹配不了
Anima 的下划线命名（`lora_unet_blocks_0_...`），因此在 Anima 上调分层完全无效。本节点修正了这点。

包含两个正式节点（菜单分类 `loaders`），外加两个 LoKr 实验节点（见文末）：

| 节点 | 用途 | 产出 |
|---|---|---|
| Anima LoRA Block Weight V2 | 运行时分层，实时生效 | 调试，不产文件 |
| Anima LoRA Block Weight Export V2 | 把分层烘焙成新 LoRA | 导出 .safetensors |

## 安装

把整个文件夹放进 ComfyUI 的 `custom_nodes` 目录，然后重启 ComfyUI：

```
ComfyUI/custom_nodes/Comfyui-Anima-lora-block-weight/
```

也可以在该目录下用 git 克隆：

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/rom0718/Comfyui-Anima-lora-block-weight.git
```

重启后，节点出现在右键菜单的 `loaders` 分类下（LoKr 实验节点在 `loaders/experimental`）。
依赖：仅需 ComfyUI 自带的 torch / safetensors，无额外第三方包。

## 一个重要的认知前提（先读）

很多人以为「DiT 模型某几层固定管构图、某几层固定管细节」。但查证 NVIDIA 官方资料后确认：
**Anima/Cosmos 是均匀堆叠的标准 DiT，28 个 block 结构完全相同，架构上没有固定的功能分区。**

也就是说，「哪段 block 管动作、哪段管色彩」这种功能分布**不是架构自带的，而是每个 LoRA 训练
出来、各不相同的**。实测也印证：画风 LoRA 的强区常在中部，角色 LoRA 的强区可能在尾部（如 26-27）。

所以本节点**不提供任何写死的功能预设**——写死的预设在 A 上对、换到 B 就错。它改用两个办法：

1. **自动分段**：按当前 LoRA 的实测指标现场分四段，而非套固定区间。
2. **双指标**：提供两种衡量 block 的角度，帮你从不同维度观察自己的 LoRA。

真正每段管什么，仍只能靠你出图实测确认——任何静态指标都只是辅助，不是答案。

## 核心概念

每个权重张量的最终缩放系数：

```
factor = 该 block 的权重 × 子模块类型系数
```

- **block 权重**：`grouped` 来自四段（`seg_1`~`seg_4`）；`per_block` 来自各 block 滑块（`blk00`~`blk27`）。
- **子模块类型系数**：`self_attn` / `cross_attn` / `mlp` / `adaln` 各一个乘数，两种模式下都生效。
- `alpha` / `dora_scale` 不缩放（缩放 down 侧一块已等效改变整体贡献）。

**四段为什么叫 `seg_1`~`seg_4`（纯编号）**：早期用过 `motion/proportion/core/detail`（绑定功能假设）
和 `weak/medium/strong/peak`（绑定强弱）。但同一套节点支持两种指标，各指标下「第几档」含义不同
甚至相反，任何含功能/强弱暗示的名字都会在某个指标下误导，故用最中性的位置编号。

## 自动分段（默认开启）

打开 `auto_segment`（默认开）后，节点读取当前 LoRA 的实测指标，把 28 个 block 分成四段自动
填进区间框。具体怎么切，由 `segment_method`（分段方式）决定，有两种：

**jenks（自然断点，默认）** —— Jenks 自然断点法，在数据**自然的缝隙**处切档：

- 让每档内部数值尽量接近、档与档之间尽量拉开（类内方差最小、类间方差最大）。
- **段大小可不均衡**，这是**如实反映 LoRA 的真实结构**——比如角色 LoRA 能量堆在尾端，
  它就会得到一个"尾端的大强档 + 几个小弱档"，告诉你主要调尾端、其他段只是微调。
- 用沙盒里 6 个真实文件验证过：jenks 的分档质量（GVF）在每个文件、每个指标上都优于均分法。

**quantile（均分）** —— 按指标值排序机械四等分：

- 每档数量均衡（约各 7 个），四个滑块灵敏度相近、手感稳。
- 但它不看数值实际差距，可能把数值差很远的 block 因为名次相邻而塞进同一档。
- 带平滑（抹孤立毛刺）和最小段长保护（避免单块碎片）。作为 jenks 的备选/退路。

> 关于 Jenks：这是制图学家 George F. Jenks 在 1967 年提出的经典一维数据分类法
> （Jenks Natural Breaks），属于公共领域的标准算法，ArcGIS 等软件也以它为默认分类法。
> 本项目直接采用该算法，不是自创。

共同点：两种方式段内都尽量连续、段间可不连续（如 `seg_4` 可能是 `16-18,24-27` 这样分开的两段）。
开关打开时四个区间框置灰（由指标和方式决定）；想手动微调，先关掉 `auto_segment`。
默认开启 + 默认 jenks，是为了新手开箱即用、且分段忠实于 LoRA 自身结构。

**关于默认组合（norm + jenks）：** 我们在多个真实 LoRA（画风 / 美学 / 角色等）上测试了
全部四种「指标 × 方式」组合，发现 **norm + jenks 是最稳健的默认**——它在测过的每个 LoRA 上
四段强度都拉得开、且从不出现"调了没反应"的空段，哪种 LoRA 上都不翻车。所以最终决定把它定为默认。

但要强调：**没有哪种组合对所有 LoRA 都最优。** 我们实测发现最佳组合因 LoRA 而异——
比如某些画风 LoRA 在 effective_rank + 均分 下四段区分更明显，而某些 LoRA 又是 effective_rank + jenks
更好。这也是为什么这两个开关保留为可自定义选项：默认给你一个最稳的起点，想针对单个 LoRA
调到最优，就把四种组合都试一遍，看哪种的四段调下去区分最明显。

## 双分段指标（segment_metric）

顶部 `segment_metric` 可切换两种指标（左右箭头），决定自动分段和染色按什么算：

- **norm（范数，默认）**：衡量「权重强弱」。
- **effective_rank（有效秩）**：衡量「权重方向丰富度」。

**为什么要两个**：实测发现这两个指标**高度负相关**——范数高的层往往能量集中在少数方向
（强而专一的变换），范数低的层往往方向分散（广泛微弱的全局调整，可能对应色彩/质感这类弥漫内容）。
单看范数会把「范数低但方向丰富」的层一律当弱忽略掉，有效秩能把它们挑出来。两者互补。

> ⚠ 这两个都是**静态指标**，只反映权重属性，**不等于功能段**。哪个指标的分段更贴近你要的
> 功能区分，必须你自己出图实测对比——提供双指标正是为了方便你做这个对比。

### 一次真实实测（供参考，方法可复用）

用 effective_rank、在一个画风 LoRA 上单变量实测（固定种子/提示词/采样器/步数/CFG，每次只把
某段权重单独降到 0.3）。**结论只适用于这一个 LoRA，仅作示范——换 LoRA、换指标，结果都会不同。**

| 段（该 LoRA 在有效秩下的分布） | 单独降到 0.3 时的画面变化 |
|---|---|
| 其中一段 | 画风总开关：笔触/上色/动作/骨相全明显变化，影响最大 |
| 另一段 | 主要改变「动作」，其余基本不变 |
| 另一段 | 控制小物体/道具完整性：手持物形变甚至崩坏，但不影响人物主体 |
| 另一段（最高频段） | 控制最高频细节：骨相几乎不变，主要是画面变干净（去高频杂质） |

价值不在记住「哪段管什么」（只对该 LoRA 成立），而在说明：effective_rank 分出来的段确实能对应
不同功能；「影响最大的段」在有效秩下未必数值最高（这正是改用纯编号的原因）；想知道你自己 LoRA
的分布，照此方法跑一遍即可。

## 子模块类型系数（w_self_attn / w_cross_attn / w_mlp / w_adaln）

前面的四段（seg_1~4）是按 **block 位置**分层——控制「哪些层」。而这四个 `w_` 系数是另一个维度，
按 **block 内部的子模块类型**分层——控制「每层里的哪一部分」。两个维度同时生效，可叠加使用。

Anima/Cosmos 的每个 block 内部，都由这四类子模块构成。用大白话解释它们各自在做什么：

- **self_attn（自注意力）**：让画面内部各个位置互相「看」对方、协调彼此关系。
  通俗说，它处理的是图像自身内部的结构与一致性。
- **cross_attn（交叉注意力）**：负责把你的**提示词文本**接进来——模型靠它在去噪过程中
  持续参考你写的 prompt。它是「文本条件」进入画面的通道。
- **mlp（前馈层）**：对每个位置的特征单独做一次非线性变换、加工。
  可以理解成每层里的「特征处理器」。
- **adaln（自适应层归一化）**：用当前去噪到第几步的「时间步」信息，去调制每一层的强度节奏。
  它不是独立模块，而是附着在上面几类上的一个调制开关。

怎么用：每个系数默认 1.0（= 原样）。把某个**调低**（如 0.7、0.5），就是**弱化该类子模块在整个
LoRA 里的贡献**；调到 0 则几乎关掉该类。调高（>1.0）会放大，但容易出伪影，上限 2.0、谨慎用。
它和四段是乘在一起的（factor = 段权重 × 该子模块系数），所以你可以「只削某段里的某类子模块」。

> ⚠ 关键提醒：**不要假设「self_attn 一定管结构、mlp 一定管画风」这种固定对应**。
> 和 block 段一样，每类子模块具体承载什么功能，是**每个 LoRA 训练出来、各不相同的**，
> 没有架构级的固定分工。实测中甚至出现过反直觉的情况——在某个 LoRA 上 mlp 反而是最强的
> 骨相开关、self_attn 对骨相几乎没影响，正好和「通用经验」相反。
> 所以这四个旋钮的真实作用，同样要靠你用「扫描定位法」在自己的 LoRA 上一个个试出来。
> 一个经验性的起点：adaln 往往没有单一明确方向（影响比较弥散），日常可优先保持 1.0，
> 先从 self_attn / cross_attn / mlp 这三个里试。

## impact 染色

逐 block 面板里每行按该 block 的指标分数染色（蓝=低 → 青 → 黄 → 红=高）。
- 用当前选的 `segment_metric`（norm 或 effective_rank），经对比度拉伸。
- 是 LoRA 固有属性，不随你拉滑块变化。
- 需先生成一次才会染色。染色数据已持久化，刷新/重载/复制节点都不会丢。

## 加载节点：Anima LoRA Block Weight V2

运行时分层、实时生效，推荐的主节点。两种控制模式（`control_mode`，顶部左右箭头）：

- **grouped（四段，默认）**：`seg_1`~`seg_4`，每段一个区间框 + 一个权重。配合自动分段最省心。
- **per_block（逐块）**：28 个 block 各一滑块，有前端 JS 时显示为紧凑图形面板，适合精细扫描定位。

### 参数

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| model / clip | — | — | 接入模型与 CLIP |
| lora_name | 下拉 | — | 选择 LoRA 文件（点开弹列表） |
| strength_model / strength_clip | float | 1.0 | LoRA 对模型 / CLIP 的总强度 |
| control_mode | 选项 | grouped | grouped 四段 / per_block 逐块 |
| auto_segment | bool | true | 自动分段（开启时区间框置灰、由指标决定） |
| segment_metric | 选项 | norm | 分段/染色指标：norm 范数 / effective_rank 有效秩 |
| segment_method | 选项 | jenks | 分段方式：jenks 自然断点（忠实结构）/ quantile 均分（手感稳） |
| seg_1_blocks ~ seg_4_blocks | str | 自动 | 四段区间（自动分段开时由指标填） |
| seg_1_weight ~ seg_4_weight | float | 1.0 | 四段各自的权重 |
| blk00 … blk27 | float | 1.0 | 逐 block 系数（per_block） |
| w_self_attn / w_cross_attn / w_mlp / w_adaln | float | 1.0 | 子模块类型系数 |
| verbose | bool | false | 控制台打印每 block 实际系数 |

**关于前端 JS 与「原生兜底」**：所有控件底层都是 ComfyUI 原生 widget，JS 只是隐藏它们、重绘成
紧凑面板。万一 JS 没加载，节点会退回原生控件直接铺开——很长但功能完全可用。若看到散乱原生滑块：
① 先硬刷新 `Ctrl+Shift+R`（最常见是缓存）；② 确认 `web/anima_block_weight.js` 存在；③ F12 看报错。

## 导出节点：Anima LoRA Block Weight Export V2

把分层缩放烘焙进新的 `.safetensors`。与加载节点互补：加载节点调试不产文件，导出节点把同一套
缩放固化成成品，供任何普通 LoRA Loader 直接加载（效果 = 你那组系数 + strength 1.0）。

实现：缩放系数乘进每个模块的 down 侧一块（kohya 的 `lora_down`、diffusers 的 `lora_A`、
LoKr 的 `lokr_w1`），up 侧 / alpha / dora_scale 不变；保留原 `__metadata__` 并追加导出记录。

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| output_name | str | anima_lora_baked | 输出文件名（自动补 .safetensors） |
| save_to | 选项 | loras | 存到 loras（可直接加载）或 output |
| overwrite | bool | false | 同名是否覆盖；false 时自动加 _1/_2 后缀 |

其余参数（control_mode / auto_segment / segment_metric / segment_method / 四段 / 子模块系数）与加载节点一致。

## 三种存储格式通吃

自动识别并支持三种 LoRA 存储格式，无需手动指定：
- **kohya**：`.lora_down.weight` / `.lora_up.weight`（最常见）
- **diffusers**：`.lora_A.weight` / `.lora_B.weight`（部分 LoRA，早期版本算不出强度，现已支持）
- **LoKr**：`.lokr_w1` / `.lokr_w2`（实验节点支持）

## 扫描定位法（找出你自己 LoRA 的真实功能分布）

功能分布因 LoRA 而异，最可靠是亲自扫一遍：

1. 固定 seed / 提示词 / 采样器 / 步数 / CFG，其它 LoRA 全禁用。
2. 用自动分段先得四段，或用 per_block 一次只动一小段。
3. 每次把一段（或一组子模块系数）降到 0.3~0.5，与「全 1.0 原图」对比：构图塌了→管结构；
   画风变了→管画风；色彩变了→管色彩。
4. 记录后回 grouped 按测出的分界微调。
5. 可切换 `segment_metric` 各扫一遍，看哪个指标分出来的段更「干净」对应某类功能。

调参建议：一次只动一组旋钮、固定种子对比；系数主要在 0.0~1.0 探索，>1.0 易出伪影（上限 2.0）；
打开 `verbose` 核对实际施加值。

## LoKr 实验版（可选，进阶）

正式版两个节点对普通 LoRA 已实测稳定。手里是 LoKr（LoRA 变体，权重存法不同）可试 `experimental`
目录里的两个实验节点（菜单 `loaders/experimental`），参数玩法和正式版一样，也能自动识别普通 LoRA。

**真实状态（重要）**：「数学上验证通过，但没有做出图实测」，所以一直保持实验版、不合并进正式节点。

- 已验过（代码在多个真实 LoKr 文件上跑过）：缩放数学正确（只乘 w1 一块、等效整体、不会算成平方）；
  影响力排序正确（与完整重建的标准答案一致）；各种存法（整块/拆分/带 DoRA）和参数类型都不崩
  （完全不读 alpha/rank，避开某些 LoKr 的 inf alpha 陷阱）。
- 没验、也不打算在此版本验：出图效果好不好；LoKr 的功能分布（要逐变量出图，工作量大，暂不做）。
- 实验分支安全加载，出错也不影响正式版。不想要就删掉整个 `experimental` 目录。
