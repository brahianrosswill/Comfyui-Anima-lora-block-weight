Anima LoRA Block Weight
==============================================================================

【关于本项目】
这个插件是我在与 Claude（Anthropic 的 AI）进行简单测试和深度讨论后，由 Claude 开发的，
而不是我本人编写的。分层逻辑、算法和代码都由 Claude 完成，我负责在真实环境中测试、
提供实验数据和反馈。UI 设计参考了 comfyUI-Realtime-Lora
（https://github.com/shootthesound/comfyUI-Realtime-Lora）的逐层滑块 + impact 染色交互思路。
特此说明，以免大家产生疑问或歧义。

【界面语言】
节点界面文字（标签、按钮、提示）会根据 ComfyUI 的语言设置（Comfy > Locale）自动在中文/英文
之间切换。注意：切换的只是显示文字，后端参数名不变，所以工作流存储、API 调用不受影响。

------------------------------------------------------------------------------
这是什么
------------------------------------------------------------------------------

为 LoRA 提供分层（block weight）控制的 ComfyUI 节点。最初是为 Anima（NVIDIA Cosmos 架构的
2B 二次元模型）开发的，现在发展成了一个更通用的「按实测指标自动分段」的分层工具。
你可以在生成时实时调整某些 block 的强度，也可以把调好的分层烘焙成一个新的 LoRA 文件。

社区现有的 LoRA 分层节点多为 Flux/SDXL 设计，正则写死了点号命名（blocks.0），匹配不了
Anima 的下划线命名（lora_unet_blocks_0_...），所以在 Anima 上调分层完全无效。本节点修正了
这点，并已在多个真实文件上验证。

包含两个正式节点（菜单分类 loaders）：

  Anima LoRA Block Weight V2         运行时分层，实时生效（调试用，不产文件）
  Anima LoRA Block Weight Export V2  把分层烘焙成新 LoRA（产出 .safetensors）

外加两个 LoKr 实验节点（loaders/experimental，可选，见文末）。

------------------------------------------------------------------------------
一个重要的认知前提（先读这段，它决定了你该怎么用）
------------------------------------------------------------------------------

很多人以为「DiT 模型的某几层固定管构图、某几层固定管细节」。但查证 NVIDIA 官方资料后确认：
Anima/Cosmos 是均匀堆叠的标准 DiT，28 个 block 结构完全相同，架构上【没有固定的功能分区】。

这意味着：「哪一段 block 管动作、哪一段管色彩」这种功能分布，【不是架构自带的，而是每个
LoRA 在训练中自己形成的，各不相同】。实测也印证了这点——画风 LoRA 的强区常在中部，而角色
LoRA 的强区可能在尾部（如 block 26-27）。

所以本节点【不提供任何写死的功能预设】（那种「这个预设削画风、那个预设保人物」的东西）。
写死的预设在 A 这个 LoRA 上对、换到 B 就错。本节点改用两个办法应对：
  1. 自动分段：按当前这个 LoRA 的实测指标，现场把它分成四段（而不是套固定区间）。
  2. 双指标：提供两种衡量 block 的角度（见下），帮你从不同维度观察自己的 LoRA。
真正每段管什么功能，仍然只能靠你出图实测确认——任何静态指标都只是辅助，不是答案。

------------------------------------------------------------------------------
核心概念
------------------------------------------------------------------------------

每个权重张量的最终缩放系数：

    factor = 该 block 的权重 × 子模块类型系数

· block 权重：grouped 模式来自四段（seg_1 ~ seg_4）；per_block 模式来自每个 block 自己的
  滑块（blk00 ~ blk27）。
· 子模块类型系数：self_attn / cross_attn / mlp / adaln 各一个乘数，两种模式下都生效。
· alpha / dora_scale 等不缩放（缩放 down 侧一块已等效改变整体贡献）。

四段为什么叫 seg_1 ~ seg_4（纯编号）：
  早期版本曾用 motion/proportion/core/detail（绑定功能假设）、又改过 weak/medium/strong/peak
  （绑定强弱）。但因为同一套节点支持两种分段指标，各指标下「第几档」的含义不同、甚至相反，
  任何含功能或强弱暗示的名字都会在某个指标下误导。所以最终用最中性的位置编号：
  seg_1 = 最靠前的一档，seg_4 = 最靠后的一档（具体哪些 block 落在哪段，由自动分段或你手填决定）。

------------------------------------------------------------------------------
自动分段（默认开启）
------------------------------------------------------------------------------

打开 auto_segment（默认就是开的）后，节点会读取当前 LoRA 的实测指标，用「分位法」把 28 个
block 分成四段，自动填进四个区间框：
· 分位法：把所有 block 按指标值排序，最低的约 1/4 归 seg_1、依次类推、最高的约 1/4 归 seg_4。
· 段内连续、段间可不连续——比如某 LoRA 的 seg_4 可能是「16-18,24-27」这样分开的两段。
· 还会做平滑（抹掉孤立毛刺）和最小段长保护（避免某段只剩一个孤立 block 的碎片）。

开关打开时，四个区间框会置灰（由指标决定，手填无效）；想手动微调区间，先关掉开关。
默认开启是为了让新手开箱即用：拖出节点、选好 LoRA，它就已经按这个 LoRA 自己分好段了。

------------------------------------------------------------------------------
双分段指标（segment_metric）
------------------------------------------------------------------------------

节点顶部的 segment_metric 可在两种指标间切换（左右箭头），它决定自动分段和染色按什么来算：

· norm（范数，默认）：衡量每个 block 的「权重强弱」。强区 = 改动幅度最大的层。
· effective_rank（有效秩）：衡量每个 block 的「权重方向丰富度」。

为什么要两个：实测发现这两个指标【高度负相关】——
  · 范数高的层，往往能量集中在少数几个方向（像是在做某种强而专一的变换）；
  · 范数低的层，往往方向分散（像是在做广泛而微弱的全局调整，可能对应色彩/质感这类弥漫内容）。
所以单看范数，会把那些「范数低但方向丰富」的层一律当成「弱」而忽略掉。有效秩能把它们挑出来。
两个指标互补，分出来的段不一样，各自可能对应不同类型的功能。

⚠ 重要：这两个都是【静态指标】，只反映权重本身的某种属性，【不等于功能段】。
到底哪个指标分出来的段更贴近你想要的功能区分，必须你自己出图实测对比——提供双指标，
正是为了方便你做这个对比，而不是替你下结论。

------------------------------------------------------------------------------
一次真实实测（供参考，方法可复用）
------------------------------------------------------------------------------

下面是用 effective_rank 指标、在一个画风 LoRA 上做的单变量实测：固定种子/提示词/采样器/步数/CFG，
每次只把某一段的权重单独降到 0.3，看画面变化。【结论只适用于这一个 LoRA，仅作示范——换个
LoRA、换个指标，结果都会不同。】

  段（该 LoRA 在有效秩下的分布）  |  单独降到 0.3 时的画面变化
  其中一段                        |  画风总开关：笔触/上色/动作/骨相全都明显变化，影响最大
  另一段                          |  主要改变「动作」，笔触/上色/骨相基本不变
  另一段                          |  控制小物体/道具的完整性：手持物会形变甚至崩坏，但不影响人物主体
  另一段（最高频段）              |  控制最高频细节：骨相几乎不变，主要是画面变干净（去除高频杂质）

这个实测的价值不在于「记住哪段管什么」（那只对这个 LoRA 成立），而在于说明：
  · effective_rank 分出来的段，确实能对应到不同功能（不是只分了强弱）；
  · 「影响最大的那段」在有效秩下未必是数值最高的段——这正是当初放弃强弱命名、改用纯编号的原因；
  · 想知道你自己 LoRA 每段管什么，照这个方法跑一遍即可。

------------------------------------------------------------------------------
impact 染色
------------------------------------------------------------------------------

逐 block 面板里，每行会按该 block 的指标分数染色（蓝=低 → 青 → 黄 → 红=高）。
· 染色用的就是当前选的 segment_metric（norm 或 effective_rank），经对比度拉伸得到。
· 它是 LoRA 的固有属性，不随你拉滑块变化（滑块=你施加的权重，染色=LoRA 本身的分布）。
· 需要先生成一次，节点才会把数据传给前端染色。染色数据已做持久化，刷新/重载/复制节点都不会丢。

------------------------------------------------------------------------------
加载节点：Anima LoRA Block Weight V2
------------------------------------------------------------------------------

运行时分层、实时生效，是推荐使用的主节点。

两种控制模式（control_mode 切换，顶部左右箭头）：
  grouped（四段，默认）：seg_1 ~ seg_4，每段一个区间框 + 一个权重。配合自动分段使用最省心。
  per_block（逐块）：28 个 block 各一个滑块，装有前端 JS 时显示为紧凑图形面板
    （方块开关 + 滑块 + 数字框 + impact 染色）。适合给你的 LoRA 做精细的扫描定位。

参数

  参数  |  类型  |  默认  |  说明
  model / clip  |  —  |  —  |  接入模型与 CLIP
  lora_name  |  下拉  |  —  |  选择 LoRA 文件（点开弹列表）
  strength_model / strength_clip  |  float  |  1.0  |  LoRA 对模型 / CLIP 的总强度
  control_mode  |  选项  |  grouped  |  grouped 四段 / per_block 逐块
  auto_segment  |  bool  |  true  |  自动分段（开启时区间框置灰、由指标决定）
  segment_metric  |  选项  |  norm  |  分段/染色指标：norm 范数 / effective_rank 有效秩
  seg_1_blocks ~ seg_4_blocks  |  str  |  自动  |  四段区间（自动分段开时由指标填）
  seg_1_weight ~ seg_4_weight  |  float  |  1.0  |  四段各自的权重
  blk00 … blk27  |  float  |  1.0  |  逐 block 系数（per_block 模式）
  w_self_attn / w_cross_attn / w_mlp / w_adaln  |  float  |  1.0  |  子模块类型系数
  verbose  |  bool  |  false  |  控制台打印每 block 实际系数

关于前端 JS 与「原生兜底」：
  所有控件底层都是 ComfyUI 原生 widget，前端 JS 只是把它们隐藏、重绘成统一紧凑面板
  （「原生控件做底座，JS 做皮肤」）。万一 JS 没加载（缓存太旧/文件缺失），节点会退回到原生
  控件直接铺开——很长、不美观，但功能完全可用、出图正常。若看到一堆散乱原生滑块：
    1) 先硬刷新浏览器 Ctrl+Shift+R（最常见原因是前端缓存）；
    2) 仍不行则确认 web/anima_block_weight.js 存在；
    3) 按 F12 看浏览器控制台报错。

------------------------------------------------------------------------------
导出节点：Anima LoRA Block Weight Export V2
------------------------------------------------------------------------------

把分层缩放烘焙进一个新的 .safetensors 文件。与加载节点互补：加载节点用于调试、不产文件；
导出节点把同一套缩放固化成成品，供任何普通 LoRA Loader 直接加载。

推荐流程：先用加载节点反复试出满意的系数 → 把同一组系数填进导出节点 → 得到成品文件。
此后加载该文件的效果 = 你那组系数 + strength 1.0，不必再挂分层节点，也便于分享。

实现：缩放系数被乘进每个模块的 down 侧一块（kohya 是 lora_down、diffusers 是 lora_A、
LoKr 是 lokr_w1），up 侧 / alpha / dora_scale 保持不变；原文件的 __metadata__ 保留并追加导出记录。

专属参数

  参数  |  类型  |  默认  |  说明
  output_name  |  str  |  anima_lora_baked  |  输出文件名（自动补 .safetensors）
  save_to  |  选项  |  loras  |  存到 loras（可直接加载）或 output
  overwrite  |  bool  |  false  |  同名是否覆盖；false 时自动加 _1/_2 后缀

其余参数（control_mode / auto_segment / segment_metric / 四段 / 子模块系数）与加载节点一致。

------------------------------------------------------------------------------
三种存储格式通吃
------------------------------------------------------------------------------

节点会自动识别并支持三种 LoRA 存储格式，无需手动指定：
· kohya 命名：.lora_down.weight / .lora_up.weight（最常见）
· diffusers 命名：.lora_A.weight / .lora_B.weight（部分 LoRA 用，早期版本算不出强度，现已支持）
· LoKr：.lokr_w1 / .lokr_w2（实验节点支持）

------------------------------------------------------------------------------
扫描定位法（找出你自己 LoRA 的真实功能分布）
------------------------------------------------------------------------------

既然功能分布因 LoRA 而异，最可靠的办法是亲自扫一遍：

1. 固定 seed、提示词、采样器、步数、CFG（控制变量），其它 LoRA 全部禁用。
2. 用自动分段先得到四段，或用 per_block 一次只动一小段。
3. 每次只把一段（或一组子模块系数）降到 0.3~0.5，与「全 1.0 原图」对比：
   构图/骨相塌了 → 该段管结构；画风/笔触变了 → 该段管画风；色彩变了 → 该段管色彩……
4. 记录结果，回到 grouped 模式按测出的分界微调。
5. 可以切换 segment_metric（norm / effective_rank）各扫一遍，看哪个指标分出来的段更「干净」
   地对应某类功能——这正是双指标的用途。

调参建议：一次只动一组旋钮、固定种子做对比；系数主要在 0.0~1.0 探索，>1.0 容易出伪影
（上限 2.0）；打开 verbose 可在控制台核对实际施加值。

------------------------------------------------------------------------------
LoKr 实验版（可选，进阶）
------------------------------------------------------------------------------

正式版两个节点对普通 LoRA 已实测稳定。如果你手里是 LoKr（LoRA 的一种变体，权重存法不同），
可以试 experimental 目录里的两个实验节点（菜单 loaders/experimental）：
· Anima LoKr Block Weight [实验性]        —— 运行时分层加载
· Anima LoKr Block Weight Export [实验性] —— 烘焙导出
参数、界面、玩法都和正式版一样，也能自动识别普通 LoRA。

这个实验版的真实状态（重要）：
  「数学上验证通过，但没有做出图实测」，所以一直保持实验版，不合并进正式节点。

  已验过的（用代码在多个真实 LoKr 文件上跑过）：
  · 缩放数学正确——只把系数乘到 w1 一侧的一块，等效整体缩放，不会算成平方；
  · 影响力排序正确——和把权重完整拼出来算的标准答案一致；
  · 各种存法（整块 / 拆分 / 带 DoRA）和各种参数类型都不会崩（完全不读 alpha/rank，
    天然避开某些 LoKr 把 alpha 设成超大值、存成 fp16 后溢出为 inf 的陷阱）。

  没验、也不打算在这个版本里验的：
  · 出图效果好不好；
  · LoKr 的功能分布——拉某个 block 画面具体会怎么变（要逐变量出图，工作量大，暂不做）。

  为什么这样安排：正式版给普通 LoRA 用、已稳定，不动它；LoKr 留作实验版，想用的人可以试，
  但心里清楚「数学对、功能图未实测」。实验分支是安全加载的，就算它出错也不影响正式版。

  如果不想要实验功能，把整个 experimental 目录删掉即可，不影响正式版。
