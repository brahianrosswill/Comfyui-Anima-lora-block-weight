==============================================================================
Anima LoRA Block Weight
==============================================================================

为 Anima（NVIDIA Cosmos 架构，2B 二次元模型）的 LoRA 提供分层（block weight）控制的 ComfyUI 节点。
支持运行时调试与烘焙导出：既能在生成时实时调整每层强度，也能把调好的分层固化成一个新的 LoRA 文件。

社区现有的 LoRA 分层节点为 Flux/SDXL 设计，其正则匹配写死了点号命名（blocks.0），无法匹配
Anima LoRA 的下划线命名（lora_unet_blocks_0_...），因此在 Anima 上调整分层完全无效。本节点修正了
这一点，经真实 Anima LoRA（28 block、896 个权重张量）验证：100% 匹配，零漏匹配。

包含两个节点（菜单分类 loaders）：

  节点  |  用途  |  产出
  Anima LoRA Block Weight  |  运行时分层，实时生效  |  调试，不产生文件
  Anima LoRA Block Weight Export  |  把分层烘焙成新 LoRA  |  导出 .safetensors

------------------------------------------------------------------------------
安装
------------------------------------------------------------------------------

将本仓库放入 ComfyUI 的 custom_nodes 目录，确保 __init__.py 直接位于该层：

    ComfyUI/custom_nodes/anima-lora-block-weight/
        ├── __init__.py
        ├── anima_lora_block_weight.py            # 加载节点：运行时分层
        └── anima_lora_block_weight_export.py     # 导出节点：烘焙成文件

重启 ComfyUI，在 loaders 分类下找到两个节点。无第三方依赖。

------------------------------------------------------------------------------
原理
------------------------------------------------------------------------------

每个权重张量的最终缩放系数：

    factor = block权重 × 子模块类型系数

· block 权重：grouped 模式来自 shallow / middle / deep 三档；per_block 模式来自 block_weights 逐 block 指定。
· 子模块类型系数：self_attn / cross_attn / mlp / adaln 各一个乘数，两种模式下均生效。
· .alpha 张量不缩放（缩放 up/down 已等效改变贡献）。

层与子模块的职责

  维度  |  取值  |  职责
  深度 · 浅层  |  block 0–8  |  全局构图、姿态、骨架、比例
  深度 · 中层  |  block 9–18  |  主体形态、语义过渡
  深度 · 深层  |  block 19–27  |  风格、笔触、纹理、上色
  类型 · self_attn  |  w_self_attn  |  图像内部空间结构、构图、骨相
  类型 · cross_attn  |  w_cross_attn  |  文本→画面的语义映射、提示词响应
  类型 · mlp  |  w_mlp  |  风格、纹理、画风特征
  类型 · adaln  |  w_adaln  |  DiT 自适应归一化调制，整体调性

上述 block 区间为 DiT 通用经验起点，非针对特定 LoRA 的实测值。建议用逐 block 模式扫描定位
你自己 LoRA 的真实分界点（见"扫描定位法"）。

------------------------------------------------------------------------------
加载节点：Anima LoRA Block Weight
------------------------------------------------------------------------------

运行时分层，实时生效，用于调试与日常使用。

参数

  参数  |  类型  |  默认  |  说明
  model / clip  |  —  |  —  |  接入模型与 CLIP
  lora_name  |  下拉  |  —  |  选择 LoRA 文件
  strength_model  |  float  |  1.0  |  LoRA 对模型的总强度
  strength_clip  |  float  |  1.0  |  LoRA 对 CLIP 的总强度
  control_mode  |  选项  |  grouped  |  grouped 三档 / per_block 逐块
  shallow_blocks / _weight  |  str / float  |  "0-8" / 1.0  |  浅层区间与系数（grouped）
  middle_blocks / _weight  |  str / float  |  "9-18" / 1.0  |  中层区间与系数（grouped）
  deep_blocks / _weight  |  str / float  |  "19-27" / 1.0  |  深层区间与系数（grouped）
  block_weights  |  多行文本  |  ""  |  逐 block 权重（per_block）
  w_self_attn / w_cross_attn / w_mlp / w_adaln  |  float  |  1.0  |  子模块类型系数
  default_weight  |  float  |  1.0  |  未指定 block 的缺省系数
  verbose  |  bool  |  false  |  控制台打印每 block 实际系数

全部系数保持默认 1.0 时，等价于普通 LoRA Loader。

per_block 文本框写法

三种写法可混用（逗号、换行、分号均可分隔），后写覆盖先写：

    纯序列：   1,1,0.5,0.5            # 依次对应 block 0,1,2,3...；不足补 default
    索引:值：  5:0.3, 27:1.2          # 仅改 block 5 和 27
    区间:值：  0-8:0.3, 19-27:1.0     # 批量设区间
    混写：     0-27:1.0, 12:0, 13:0   # 全设 1.0，再单独关掉 12、13

区间写法（shallow/middle/deep_blocks）支持 all / 0-8 / 0,3,5 / 0-8,19-27。
区间重叠时优先级 deep > middle > shallow。

常用配方

  目标  |  shallow  |  middle  |  deep  |  类型系数
  保画风、弱化构图骨相  |  0.3  |  0.7  |  1.0  |  全 1.0
  保构图骨相、弱化画风  |  1.0  |  0.7  |  0.3  |  全 1.0
  仅削体型/骨相  |  1.0  |  1.0  |  1.0  |  w_self_attn=0.5
  抑制抢提示词/概念污染  |  1.0  |  1.0  |  1.0  |  w_cross_attn=0.5
  加强画风  |  1.0  |  1.0  |  1.0  |  w_mlp=1.2

------------------------------------------------------------------------------
导出节点：Anima LoRA Block Weight Export
------------------------------------------------------------------------------

把分层缩放烘焙进一个新的 .safetensors 文件。与加载节点互补：加载节点用于调试、不产文件；
导出节点把同一套缩放固化成成品，供任何普通 LoRA Loader 直接加载。

推荐流程：先用加载节点反复试出满意的系数 → 把同一组系数填进导出节点 → 得到成品文件。
此后加载该文件的效果 = 你那组系数 + strength 1.0，不必再挂分层节点，也便于分享。

实现

缩放系数 factor（同上）被乘进每个模块的 lora_down 权重（单边缩放，数学等价于运行时
tensor*factor），lora_up 与 alpha 保持不变；原 LoRA 的 __metadata__ 保留并追加导出记录。
经真实文件验证：缩放比精确、up/alpha 不变、文件结构完整。

专属参数

  参数  |  类型  |  默认  |  说明
  output_name  |  str  |  anima_lora_baked  |  输出文件名（自动补 .safetensors）
  save_to  |  选项  |  loras  |  存到 loras（可直接加载）或 output
  overwrite  |  bool  |  false  |  同名是否覆盖；false 时自动加 _1/_2 后缀

其余参数（control_mode / 三档 / block_weights / 四个类型系数 / default_weight）与加载节点完全一致。

------------------------------------------------------------------------------
扫描定位法（找出你 LoRA 的真实分界）
------------------------------------------------------------------------------

逐 block 模式可用于为你的 LoRA 绘制"功能地图"：

1. 固定 seed、提示词、采样器、步数、CFG（控制变量）。
2. 用 per_block，一次只关一小段：
       0-27:1.0, 0-3:0      # 只关 0~3
       0-27:1.0, 4-7:0      # 只关 4~7
       ...                  # 直到 24-27
3. 每段对比"全 1.0 原图"：关掉后构图塌了 → 该段管骨架；画风变了 → 该段管画风。
4. 记录结果，回到 grouped 模式按测出的真实分界设三档，日常高效。

调参建议：一次只动一组旋钮、固定种子做对比；系数主要在 0.0–1.0 探索，>1.0 易出伪影（上限 2.0）；
打开 verbose 核对实际施加值。
