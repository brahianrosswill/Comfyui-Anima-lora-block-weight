==============================================================================
Anima LoRA Block Weight
==============================================================================

【关于本项目】
这个插件是我在与 Claude（Anthropic 的 AI）进行简单测试和深度讨论后，由 Claude 开发的，
而不是我本人编写的。功能地图、分层逻辑和代码都由 Claude 完成，我负责在真实环境中测试、
提供实验数据和反馈。UI 设计参考了 comfyUI-Realtime-Lora
（https://github.com/shootthesound/comfyUI-Realtime-Lora）的逐层滑块 + impact 染色交互思路。
特此说明，以免大家产生疑问或歧义。

【界面语言】
节点界面文字（按钮、提示）会根据 ComfyUI 的语言设置（Comfy > Locale）自动在中文/英文之间切换；
后端描述为中英双语并排。

------------------------------------------------------------------------------

为 Anima（NVIDIA Cosmos 架构，2B 二次元模型）的 LoRA 提供分层（block weight）控制的 ComfyUI 节点。
支持运行时调试与烘焙导出：既能在生成时实时调整每层强度，也能把调好的分层固化成一个新的 LoRA 文件。

社区现有的 LoRA 分层节点为 Flux/SDXL 设计，其正则匹配写死了点号命名（blocks.0），无法匹配
Anima LoRA 的下划线命名（lora_unet_blocks_0_...），因此在 Anima 上调整分层完全无效。本节点修正了
这一点，经真实 Anima LoRA（28 block、896 个权重张量）验证：100% 匹配，零漏匹配。

包含两个节点（菜单分类 loaders）：

  节点  |  用途  |  产出
  Anima LoRA Block Weight  |  运行时分层，实时生效  |  调试，不产生文件
  Anima LoRA Block Weight Export  |  把分层烘焙成新 LoRA  |  导出 .safetensors

------------------------------------------------------------------------------
安装
------------------------------------------------------------------------------

方式一：git clone（推荐）

    cd ComfyUI/custom_nodes
    git clone https://github.com/rom0718/Anima-lora-block-weight.git

方式二：手动下载

下载本仓库 zip，解压到 ComfyUI 的 custom_nodes 目录，确保 __init__.py 直接位于该层：

    ComfyUI/custom_nodes/Anima-lora-block-weight/
        ├── __init__.py
        ├── anima_lora_block_weight_v2.py         # 加载节点：四段 / 逐block滑块 + impact染色
        ├── anima_lora_block_weight_export.py     # 导出节点：烘焙成文件
        ├── web/
        │   └── anima_block_weight.js             # 前端面板（滑块/染色/中英切换）
        └── experimental/                          # 可选：LoKr 实验分支

两种方式都需要：重启 ComfyUI，然后硬刷新浏览器（Ctrl+Shift+R）清前端缓存
（带前端 JS 的节点更新后必须清缓存，否则可能仍用旧 JS）。

重启后在 loaders 分类下找到节点。无第三方依赖。

更新插件：cd ComfyUI/custom_nodes/Anima-lora-block-weight && git pull ，然后重启 + 硬刷新。

原理
------------------------------------------------------------------------------

每个权重张量的最终缩放系数：

    factor = block权重 × 子模块类型系数

· block 权重：grouped 模式来自实测的四段（motion / proportion / core / detail）；
  per_block 模式来自每个 block 自己的滑块（blk00–blk27）。
· 子模块类型系数：self_attn / cross_attn / mlp / adaln 各一个乘数，两种模式下均生效。
· .alpha 张量不缩放（缩放 up/down 已等效改变贡献）。

实测功能地图（V2 默认，来自真实消融实验）

与 DiT 通用经验不同，V2 的四段来自对真实 Anima 画风 LoRA 的固定种子单变量消融实验。
注意这些是有段间串扰的"倾向"，不是干净分离——每段本质是"先验服从度旋钮"，而非"画笔"。

  维度  |  取值  |  实测倾向
  seg_motion  |  block 0–11  |  动作 / 体型尺寸的提示词服从度
  seg_proportion  |  block 12–14  |  体型比例 / 敦实度的先验服从度
  seg_core  |  block 15–18  |  LoRA 核心表达段（骨相+比例+材质，信息密度最高）
  seg_detail  |  block 19–27  |  全局精修（饱和度 / 纯度等表层属性）
  类型 · self_attn  |  w_self_attn  |  细节内容 + 少量骨相 + 少量画风
  类型 · cross_attn  |  w_cross_attn  |  对画风影响最小，最适合安全微调
  类型 · mlp  |  w_mlp  |  对骨相影响最大（同时带画风）
  类型 · adaln  |  w_adaln  |  整体性影响，无单一方向

⚠ 以上倾向在画风 LoRA 上验证过、看起来是架构级特性（三个差异很大的 LoRA 表现一致），
但你的具体 LoRA 可能不同。内置的 impact 染色（per_block 面板）会计算每个 block 的权重 L2 范数，
让你一眼看到自己 LoRA 真正的高能 block；建议用逐 block 模式扫描确认（见"扫描定位法"）。
完整实测细节参见下方"实测案例"。

加载节点：Anima LoRA Block Weight V2
------------------------------------------------------------------------------

运行时分层，实时生效。V2 是推荐使用的主节点，相比 V1 增加了基于实测的四段划分、
逐 block 图形化滑块面板和 impact 染色。

三种控制模式（control_mode 切换）：

  grouped（四段，默认）：基于实测功能地图的四段，每段区间可自定义
    seg_motion     默认 0-11   动作 / 体型尺寸的提示词服从度
    seg_proportion 默认 12-14  体型比例 / 敦实度的先验服从度
    seg_core       默认 15-18  LoRA 核心表达段（骨相+比例+材质，信息密度最高）
    seg_detail     默认 19-27  全局精修
  per_block（逐块滑块）：28 个 block 各一滑块。装有前端 JS 时显示为紧凑图形面板
    （方块开关 + 滑块 + 数字框 + impact 染色）。

impact 染色：
  per_block 面板中，每行按该 block 的 impact 分数染色（蓝=低 → 青 → 黄 → 红=高）。
  impact 由运行时计算每个 block 的 LoRA 权重 L2 范数、经对比度拉伸得到，反映该 block
  在这个 LoRA 里的相对重要性。要点：
  · impact 是 LoRA 固有属性，不随你调滑块变化（滑块=你施加的权重，impact=LoRA 本身）。
  · 采用对比度拉伸（[最弱,最强] 拉到 [蓝,红]），表达"该 LoRA 内部相对重要性"，非绝对强度。
  · 需先生成一次，节点才会把 impact 数据传给前端染色。

关于前端 JS 与"原生兜底"：
  本节点所有控件（strength、四段、w_、verbose、逐 block）底层都是 ComfyUI 原生 widget；
  前端 JS 只是把它们隐藏、重绘成统一紧凑面板（"原生控件做底座，JS 做皮肤"）。
  正常安装下 JS 随节点自动加载，你看到的就是紧凑面板。
  万一 JS 没加载成功（缓存太旧 / 文件缺失 / 前端不兼容），节点会退回到所有原生控件直接铺开：
  strength / 四段区间与权重 / w_ / verbose / 28 个 blk 滑块全部一次性显示（不再按模式隐藏无关项），
  节点会很长、不美观——但功能完全可用，值正常传递、出图正常，只是失去紧凑布局、染色和按模式显隐。
  若你看到的是一堆散乱原生滑块而非紧凑面板：
    1) 先硬刷新浏览器 Ctrl+Shift+R（最常见原因是前端缓存）；
    2) 仍不行则确认 web/anima_block_weight.js 文件存在；
    3) 按 F12 打开浏览器控制台查看报错。

参数

  参数  |  类型  |  默认  |  说明
  model / clip  |  —  |  —  |  接入模型与 CLIP
  lora_name  |  下拉  |  —  |  选择 LoRA 文件
  strength_model / strength_clip  |  float  |  1.0  |  LoRA 对模型 / CLIP 的总强度
  control_mode  |  选项  |  grouped  |  grouped 四段 / per_block 逐块滑块
  seg_motion_blocks / _weight  |  str / float  |  "0-11" / 1.0  |  动作段区间与系数
  seg_proportion_blocks / _weight  |  str / float  |  "12-14" / 1.0  |  比例段区间与系数
  seg_core_blocks / _weight  |  str / float  |  "15-18" / 1.0  |  核心段区间与系数
  seg_detail_blocks / _weight  |  str / float  |  "19-27" / 1.0  |  精修段区间与系数
  blk00 … blk27  |  float  |  1.0  |  逐 block 系数（per_block；JS 会美化成面板）
  w_self_attn / w_cross_attn / w_mlp / w_adaln  |  float  |  1.0  |  子模块类型系数
  verbose  |  bool  |  false  |  控制台打印每 block 实际系数

常用配方（grouped 四段）

  目标  |  motion  |  proportion  |  core  |  detail  |  类型系数
  让体型更听提示词  |  0.5  |  1.0  |  1.0  |  1.0  |  全 1.0
  锁住比例不被提示词改  |  1.0  |  调低=更回归先验  |  1.0  |  1.0  |  全 1.0
  大幅改造、温和化画面  |  1.0  |  1.0  |  0.5-0.7  |  1.0  |  全 1.0
  仅削骨相（接受少量画风损失）  |  1.0  |  1.0  |  1.0  |  1.0  |  w_mlp=0.7
  抑制抢提示词 / 概念污染  |  1.0  |  1.0  |  1.0  |  1.0  |  w_cross_attn=0.5


------------------------------------------------------------------------------
导出节点：Anima LoRA Block Weight Export V2
------------------------------------------------------------------------------

把分层缩放烘焙进一个新的 .safetensors 文件。与加载节点互补：加载节点用于调试、不产文件；
导出节点把同一套缩放固化成成品，供任何普通 LoRA Loader 直接加载。

推荐流程：先用加载节点反复试出满意的系数 → 把同一组系数填进导出节点 → 得到成品文件。
此后加载该文件的效果 = 你那组系数 + strength 1.0，不必再挂分层节点，也便于分享。

实现

缩放系数 factor（同上）被乘进每个模块的 lora_down 权重（单边缩放，数学等价于运行时
tensor*factor），lora_up 与 alpha 保持不变；原 LoRA 的 __metadata__ 保留并追加导出记录。
经真实文件验证：缩放比精确、up/alpha 不变、文件结构完整。

专属参数

  参数  |  类型  |  默认  |  说明
  output_name  |  str  |  anima_lora_baked  |  输出文件名（自动补 .safetensors）
  save_to  |  选项  |  loras  |  存到 loras（可直接加载）或 output
  overwrite  |  bool  |  false  |  同名是否覆盖；false 时自动加 _1/_2 后缀

其余参数（control_mode / 三档 / block_weights / 四个类型系数 / default_weight）与加载节点完全一致。

------------------------------------------------------------------------------
扫描定位法（找出你 LoRA 的真实分界）
------------------------------------------------------------------------------

逐 block 模式可用于为你的 LoRA 绘制"功能地图"：

1. 固定 seed、提示词、采样器、步数、CFG（控制变量）。
2. 用 per_block，一次只关一小段：
       0-27:1.0, 0-3:0      # 只关 0~3
       0-27:1.0, 4-7:0      # 只关 4~7
       ...                  # 直到 24-27
3. 每段对比"全 1.0 原图"：关掉后构图塌了 → 该段管骨架；画风变了 → 该段管画风。
4. 记录结果，回到 grouped 模式按测出的真实分界设三档，日常高效。

调参建议：一次只动一组旋钮、固定种子做对比；系数主要在 0.0–1.0 探索，>1.0 易出伪影（上限 2.0）；
打开 verbose 核对实际施加值。

实测案例：Anima LoRA 真实功能地图
------------------------------------------------------------------------------

为说明"理论默认值仅是起点"，下方记录一次完整的扫描定位法实测。结论仅适用于被测的那个 LoRA，
但它体现了一个普遍现象：在 Anima/Cosmos 架构上，多个理论默认值可能整体偏移。其它 LoRA 请自行
扫描验证。

测试方法

控制变量：固定 seed / 提示词 / 采样器 / 步数 / CFG；其它 LoRA 全部禁用。每次只动一个旋钮，
其余保持 1.0，与全 1.0 基线对图对比。

实测结果

按 block 深度：

  段  |  实测功能  |  理论默认
  shallow 0-8  |  动作一致性：手势、表情、脚趾朝向等微观行为  |  构图/骨相 ✗
  middle 9-18  |  核心信息层：骨相 + 大部分画风 + 部分构图  |  过渡 ✗
  deep 19-27  |  少量画风细节/笔触  |  主画风 ✗

按子模块类型：

  旋钮  |  实测功能  |  理论默认
  w_self_attn  |  细节内容 + 少量骨相 + 少量画风  |  骨相/姿态 ✗
  w_mlp  |  骨相影响最大，同时也影响画风（最强单一维度开关）  |  画风 ✗
  w_cross_attn  |  对画风影响最小，少量修改细节（安全微调工具）  |  提示词响应 ⚠
  w_adaln  |  整体性影响：所有维度都会动，无专门方向  |  整体调性 ✓

反直觉发现：w_mlp 反而是该 LoRA 上最强的骨相开关；w_self_attn 在该 LoRA 上对骨相影响很小。
这与"self_attn 管空间结构"的通用经验相反，但可由训练数据的特征分布解释——当骨相与画风被同时
训练进同一段 block 的同一类子模块（此处为 middle 段的 mlp）时，单调 mlp 会同时牵动两者。

基于该地图的实战配方

  想要的效果  |  配置
  削骨相，能接受少量画风损失  |  w_mlp=0.7，其他 1.0
  大幅改造，让画面"温和化"  |  seg_core_weight=0.5-0.7，其他 1.0
  只动一点点细节，画风几乎不变  |  w_cross_attn=0.7，其他 1.0
  想保留极致 LoRA 风格  |  全 1.0，靠提示词或负面词调节具体元素
  削动作夸张程度但保留体型  |  seg_motion_weight=0.5-0.7，其他 1.0

重要现实结论

该 LoRA 上"完全分离画风和骨相"做不到——它们物理上被训练在同一段（middle）的同一类子模块（mlp）
里。这不是节点的限制，而是 LoRA 训练结果。期待"留画风去骨相"的策略不应追求干净分离，而应寻找
可接受的折中点（例如骨相收 60%、画风损失 20% 的甜点位）。

给其他 Anima LoRA 用户的建议

上述地图不是 Anima LoRA 的通用配置，每个 LoRA 仍会有差异。但相比 DiT 通用经验默认值，它可能
更贴近 Anima 架构的实际行为，可作为扫描定位的优先怀疑方向：

· 优先怀疑 middle 段是核心信息层（而非 shallow）
· 优先怀疑 mlp 与 self_attn 的职责可能与 DiT 通用经验相反
· w_adaln 通常无专门方向，可优先保持 1.0
· w_cross_attn 通常是最安全的"轻量微调档"

仍建议你按扫描定位法亲自跑一遍，得到属于你自己 LoRA 的精确地图。
